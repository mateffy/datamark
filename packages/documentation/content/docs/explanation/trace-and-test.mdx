Every format created with `datamark()` supports two introspection methods: `trace()` and `test()`.

## trace()

`trace(content)` runs your parse generator against a real document and records every step:

```typescript
const trace = PlanFormat.trace(markdown);

for (const step of trace.steps) {
  console.log(step.combinator); // "frontmatter"
  console.log(step.matched); // { id: "plan-001" }
  console.log(step.consumed); // raw nodes
  console.log(step.region); // { start: { line: 1, column: 0 }, end: { ... } }
}
```

This is invaluable when a format stops working on a new document. You can see exactly which combinator failed and what was left unconsumed.

## test()

`test()` validates all configured examples against your parse generator and schema:

```typescript
const PlanFormat = datamark({
  schema: z.object({ id: z.string() }),
  examples: [
    {
      text: "---\nid: a\n---\n# Title",
      data: { id: "a" },
    },
  ],
  *parse(doc) {
    // ...
  },
});

const result = PlanFormat.test();
console.log(result.passed); // true or false
console.log(result.failures); // [] if passed
```

<Callout type="info">
  Examples with both `text` and `data` are checked for exact output equality. Examples with only `text` are checked for parseability.
</Callout>

## Using test() in CI

Add a test script to your project:

```typescript
import { PlanFormat } from "./formats/plan";

if (import.meta.main) {
  const result = PlanFormat.test();
  if (!result.passed) {
    for (const f of result.failures) {
      console.error(`Example ${f.exampleIndex} failed: ${f.error}`);
    }
    process.exit(1);
  }
  console.log("All format examples passed.");
}
```

This keeps your format definitions self-documenting and self-testing.
