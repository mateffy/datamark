This example parses a changelog with version headings, optional release dates, and bullet lists of changes. It has no frontmatter — all data lives in the heading text and list items. AST utilities (`inlineText`, `textContent`) handle extraction; stringify primitives (`heading`, `list`) handle reconstruction.

<Callout type="info">
  For production use, the **`@datamark/changelog`** package provides four battle-tested, ready-to-use changelog formats with full Zod schemas, built-in examples, and comprehensive tests.
</Callout>

<Example name="changelog" />

```typescript
const result = ChangelogFormat.parse(changelogMarkdown);
console.log(result.versions[0].version); // "1.2.0"
console.log(result.versions[0].date);     // "2026-06-15"
console.log(result.versions[0].changes.length); // 3
```

## Key concepts

- **No frontmatter** — the format uses heading text as the primary data source
- **Regex parsing** — heading text is split into version and date with a simple regex
- **`textContent()`** from the Parse SDK extracts plain text from each list item
- **`heading()`** and **`list()`** from `datamark/stringify` reconstruct the markdown cleanly

## `@datamark/changelog` package

For real-world projects, use the predefined formats from **`@datamark/changelog`** instead of writing your own:

```typescript
import { keepAChangelog } from "@datamark/changelog";
// or: conventionalChangelog, githubReleases, commonChangelog

const data = keepAChangelog.parse(markdown);
const md   = keepAChangelog.stringify(data);
```

### Available formats

| Format | Style | Heading pattern |
|--------|-------|-----------------|
| `keepAChangelog` | [Keep a Changelog](https://keepachangelog.com/) | `## [1.0.0] - 2024-01-01` |
| `conventionalChangelog` | Angular / conventional-commits | `## [1.0.0](compare-url) (2024-01-01)` |
| `githubReleases` | GitHub Releases | `## v1.0.0 (2024-01-01)` |
| `commonChangelog` | [Common Changelog](https://common-changelog.org/) | `## [1.0.0] - 2024-01-01` with summaries |

Each format returns a typed `Changelog` object with `releases`, `categories`, `changes`, extracted `links`, `images`, `references`, and `commits`.
